# Code of Conduct

This project follows the [Contributor Covenant Code of Conduct, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

## Summary

We pledge to make participation in the solution-intelligence-graph-adapter community a respectful experience for everyone, regardless of background or identity. We commit to acting in ways that contribute to a welcoming, diverse, and healthy community.

Examples of behavior that contribute to a positive environment:

- Demonstrating empathy and kindness toward other people
- Being respectful of differing opinions, viewpoints, and experiences
- Giving and gracefully accepting constructive feedback
- Accepting responsibility, apologizing to those affected by our mistakes, and learning from the experience
- Focusing on what is best for the overall community

The full enumeration of expected behaviors, unacceptable behaviors, enforcement responsibilities, scope, and the four-tier enforcement guidelines is the canonical Contributor Covenant 2.1 text, linked above. This project adopts that text in full and by reference.

## Enforcement

Reports of behavior that may violate this Code of Conduct should be sent privately to the project maintainers via the same channel documented in [`SECURITY.md`](./SECURITY.md) for security disclosures:

- **GitHub Security Advisories** for the solution-intelligence-graph-adapter repository, OR
- The maintainer email listed in `SECURITY.md`

All reports will be reviewed and investigated promptly and fairly. The project maintainers are obligated to respect the privacy and security of the reporter.

## Scope

This Code of Conduct applies within all community spaces (issues, pull requests, discussions, code review, project-affiliated chat) and also applies when an individual is officially representing the project in public spaces.
